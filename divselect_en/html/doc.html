Divselect

This plugin is primarily a HTML <select> replacement. The plugin is flexible and customizable, so it can be used for other purposes, for example a dropdown menu also can be implemented with this.
Engaging on a select element, it replaces this (by default) to a structure built with div elements, that operation just like the original select. It places a hidden input inside the structure for post the selected value. There is a CSS file for the plugin that used for customize the display of the element.
The plugin possess all the comfort functions that the original select has, such as the stepping with cursor keys, inline search with character key presses, etc. See also in the keyboard controlling part.

  • Current version: 1.0.0
  • Dependencies jQuery, jQuery UI position
  • Required: jQuery 1.6+, jQuery UI position 1.8+
  • Browsers: IE7+, FF4+, Chrome, Opera11+, (IE6 if disable the keyboard control)
  • Download:
    • full package (zip)
    • jquery.divselect-1.0.0.js (development version)
    • jquery.divselect-1.0.0.min.js (compressed version)
    • divselect.css (basic CSS file)

With a jquery version below jQuery 1.6 some functions won't be operate appropriate.

Usage

Insert the following files:

  • jQuery 1.6+
  • jQuery UI position 1.8+
  • The plugin: jquery.divselect.js
  • CSS file: divselect.css

Example (supposed to be a HTML5 code): 

	<link rel="stylesheet" href="css/divselect.css" />
	<script src="js/jquery.js"></script>
	<script src="js/jquery-ui.position.js"></script>
	<script src="js/jquery.divselect.js"></script>

Create a select element in the HTML code. Here some attributes affect the divselect behavior, see here.
Important: the value attributes of the option tags have to be unique. 

	<select id="example" name="webprog">
		<option value="html">HTML</option>
		<option value="css">CSS</option>
		<option value="js">JavaScript</option>
		<option value="php">PHP</option>
		<option value="sql">SQL</option>
	</select>

And after the DOM loaded, we run the plugin on the select element (of course you can use any selector): 

	<script>
		$(document).ready(function(){
			$("#example").divselect();
		});
	</script>

More usage examples are in the demo pages.

Properties

You can use properties to configure the divselect, which can be passed to the divselect() method in an object at the initialization: 

	$("#example").divselect({
		cssprefix : 'divselect-sort',
		showeffect: function($elem){
			$elem.slideDown();
		},
		...
	});

The setting, getting and modifying of the parameters is similar to the jQuery UI widgets same functionality. The properties can be modified in runtime with the option or the regenerate method (methods are in here). The defined properties are the following ones:

Static properties (can be modified in runtime with the regenerate method):

Events

Some special events of the divselect event handler functions can be attached in this way: 

	// initializing:
	$("#example").divselect({
		change : function(event, value, oldvalue){
			// event handler
		},
		...
	});
	
	// in runtime:
	$("#example").on("change.divselect", function(event, value, oldvalue){
		// event handler
	});

Every event belongs to the divselect namespace, we have to use this in the attachment.
The default event handler is az empty function for all of the special events. If we set event handlers with both of the above methods, the initializing form will run first. Event handler can be attached for these events:

Event Trigger Parameters
create It is triggered after the select element replaced by the divselect structure.
It can be set only in the initialization! 		event
select It is triggered when a value selected in the list. event, value
change It is triggered when the selected value changed (a special case of the above). event, value, oldvalue
open It is triggered when the list opened. event
close It is triggered when the list closed. event
The parameters:
event (object): the jQuery Event object,
value (string): the value attribute of the selected option.
oldvalue (string): in the change event it is the value attribute of the previously selected option.
Furthermore, in the event handler the this by default refers to the actually divselect element (most outer div as a DOM object).

Methods

With the divselect methods informations can be queried and the element can be manipulated in runtime. The call of the methods: 

	$("#example").divselect("methodname", parameters);

The methods that are chainable, return the jQuery object which we have engaged the method. On this, we can call the method again, so several methods can be call in one sequence. E.g: 

	$("#example").divselect("select", "name").divselect("setreadonly");

add With this method we can add an option or more options to the list in runtime.

Parameters:
1. (string|object|array) the describer element of the option(s)
2. (string; default value: "before") this is "before" or "after" depends on we want to insert the option(s) before or after the element determined by the third parameter

If we don't pass the third parameter, in the "before" case the new option will be inserted at the end of the list, in the case of "after", it will be inserted to the beginning of the list. selected (boolean; default value: false)
readonly (boolean; default value: false)
disabled (boolean; default value: false)
If the selected is true, that results the new value will be the selected element.

Give the option with HTML code:
var html = '<option value="new" disabled="disabled">New</option>';
 We have to write into the HTML code the occurrent inner elements, like the checkbox in the above example, because we cannot change the optiontags parameter in runtime.

HTML attributes

The divselect parse some attributes of the select, optgroup and option element, and automatically set a property by them. The size and multiple attributes of the select element will be ignored. The divselect always one-line independent of the size attribute, and it only allow one element selection by default. We can find examples for selecting more than option in the demos.
The parsed attributes and their effects:

HTML tag Attribute Effect
select name This name attribute will be given to the hidden input element, that is to be passing the selected value.
select readonly If this has been set, the select state will be readonly.*
This attribute isn't valid in HTML5, but we can use the appropriate method for the same effect: $("#example").divselect("setreadonly");
select disabled If this has been set, the select state will be disabled.*
optgroup label The value of this attribute will be the optgroup label, more exactly this will replace the {optgroup.label} placeholder in the template. Furthermore, this is a unique id for the optgroups.
option value If the option is selected, this value will be in the hidden input value attribute. Furthermore, this is a unique id for the options.
option selected The option which has this attribute, will be the selected element by default. If it isn't set on any option, the first will be the selected.
option readonly If this has been set, the option state will be readonly.*
This attribute isn't valid in HTML5, but we can use the appropriate method for the same effect: $("#example").divselect("setreadonly", "php");
option disabled If this has been set, the option state will be disabled.*
* If the select is in readonly state, the selected option cannot be changed, if an option is readonly, the form elements inside this cannot be changed (more exactly the click event on the elements that match to the unbubble selector won't fire the default event handler).
If the select is in disabled state, the selected value won't be passed when the form posted, and the element doesn't respond to any user interaction. If an option is disabled, the option cannot be selected.

Template

The htmltemplate property describe the HTML structure of the divselect element. This is changable because the CSS code modification doesn't always enough for the purposed design, sometimes we have to change the HTML structure. Additionally, in this way is easy to add custom elements into the divselect. The template looks like this by default: 

	<div class="{cssprefix}">
		<input class="{cssprefix}-input" type="hidden" name="{select.name}" value="{select.val}" />
		<div class="{cssprefix}-selector {select.readonly} {select.disabled}">{select.defhtml}</div>
		<div class="{cssprefix}-menu">
			{block}
				{optgroup_start}
					<div class="{cssprefix}-group">
						<div class="{cssprefix}-grouptitle">{optgroup.label}</div>
				{/optgroup_start}
				{optgroup_end}
					</div>
				{/optgroup_end}
				{optiontags}
					<div class="{cssprefix}-item {option.readonly} {option.disabled}">{option.html}</div>
				{/optiontags}
			{/block}
		</div>
		<div class="{cssprefix}-background"></div>
	</div>

The placeholders format is {...}, these are replaced to the appropriate value, or control the HTML structure building. Only the placeholders there are in the above template are defined, other placeholders have no effect. You should modify the template only if this is necessary, and only with maintain the hierarchy! Some elements of a special CSS class have special functions:

{cssprefix} : this is the conatiner element, this has to wrap the whole structure.
{cssprefix}-input : this form element contain the selected value.
{cssprefix}-selector : by clicking this element, the option list will open.
{cssprefix}-menu : the option list container.
{cssprefix}-item : an element of the list (one option), by clicking this the option will be selected.
{cssprefix}-background : by clicking this element, the option list will close.

The default functions that has been binded to the click event can be disable with set true the disabledropdown, disableselect and disableclose properties. After this we can overwrite the base functionality by binding new event handlers to this elements.
